# MU × SK하이닉스 시그널 봇 (Signal_Bot)

마이크론(MU)과 SK하이닉스(000660.KS)의 매수/매도 시그널을 계산하고, 간단한 웹뷰
대시보드로 보여주는 봇이다. [상위 페어 분석](../README.md)의 **기본 전략**(리드-래그 모멘텀 +
스프레드 평균회귀)과 **추가 전략 7종**을 한 번에 산출한다.

면책: 연구·교육용 보조 지표이며 투자 권유가 아니다. 모든 신호는 참고용이고, 인샘플 특성과
레짐 의존성으로 실제 투자에는 상당한 리스크가 따른다.

## 구성

| 파일 | 역할 |
|---|---|
| `signal_bot.py` | 데이터 수집(yfinance) + 시그널 계산 → `signals.json` / `signals.js` 출력 |
| `index.html` | 정적 웹뷰 대시보드 (JSON을 읽어 카드/표로 렌더) |
| `signals.json` | 봇이 생성한 최신 시그널 데이터 |
| `signals.js` | `file://`로 열 때 쓰는 JS 래퍼(브라우저가 로컬 `fetch(json)`을 차단하기 때문) |

## 실행

```bash
pip install numpy pandas scipy statsmodels yfinance

# 실데이터 3년으로 시그널 생성
python signal_bot.py --years 3 --outdir .

# 네트워크 없이 로직만 검증(합성 데이터)
python signal_bot.py --mock --outdir .
```

## 웹뷰로 보기

봇 실행 후 `index.html`을 연다. 두 가지 방법:

- **간단히**: `index.html` 파일을 브라우저로 직접 연다. (`signals.js`를 통해 데이터가 들어간다.)
- **로컬 서버**: `python -m http.server 8011` 실행 후 `http://localhost:8011/` 접속.
  (`signals.json`을 `fetch`로 읽는다.)
- **GitHub Pages**: 이 폴더를 Pages로 게시하면 `index.html`이 그대로 대시보드가 된다.

## 시그널 구성

### 기본 전략

- **A. 리드-래그 모멘텀 (MU 선행)** — MU 전일 수익률이 ±1σ 초과 + `+1일` 리드-래그 상관
  유의(p<0.05)면 하이닉스 익일 BUY/SELL. 미국장이 한국장 이후 열리는 시차 구조를 이용.
- **B. 스프레드 평균회귀 (공적분 유효 시)** — z>+2 하이닉스 매도+MU 매수 / z<-2 하이닉스
  매수+MU 매도 / |z|<0.5 청산 / |z|>3 진입 금지.

### 추가 전략 7종

| # | 전략 | 무엇을 보나 |
|---|---|---|
| 1 | 3사 확장 페어 (MU·하이닉스·삼성) | 하이닉스-삼성 상대강도로 HBM 디커플링 탐지 |
| 2 | 장비·소재 공급망 스프레드 (→한미반도체) | 메모리(t-1)→장비(t) 시차 전파 유효성 |
| 3 | 환율 오버레이 (USD/KRW) | 환율 급변·원화 강약으로 모멘텀 신뢰도/수급 가중 조정 |
| 4 | 변동성 타겟팅 / 포지션 사이징 | 실현변동성 역가중 포지션 배수(목표 30%) |
| 5 | 이벤트 드리븐 오버레이 | MU 실적 ±3일 진입 게이트 ON/OFF |
| 6 | 레짐 스위칭 (60일선) | 추세장/혼조/약세장에 따라 A·B 비중 배분 |
| 7 | 옵션 기반 표현 | 공매도 제약을 풋/콜 스프레드로 우회 |

각 전략은 데이터가 있으면 실제 값을 계산하고, 보조 종목(삼성·한미반도체) 데이터가
없으면 해당 전략만 비활성("N/A") 처리한 뒤 봇은 계속 동작한다.

## 한계

- 모든 임계치(z=±2, ±1σ, 목표변동성 30% 등)는 인샘플 기준이며 워크포워드로 보정해야 한다.
- 슬리피지·대차비용·세금·공매도 제약은 반영되지 않는다.
- 공적분·상관 관계는 메모리 사이클·HBM 수요 충격 같은 레짐 전환에서 깨질 수 있다.
- 실적 캘린더 등 일부 보조 데이터는 yfinance 제공 여부에 따라 누락될 수 있다.
