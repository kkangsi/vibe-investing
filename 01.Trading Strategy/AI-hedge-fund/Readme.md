# AI 헤지펀드 (ai-hedge-fund) 분석 리포트

## 개요

`ai-hedge-fund`는 GitHub에서 5만 개가 넘는 스타를 받으며 큰 주목을 받은 오픈소스 프로젝트로, [virattt](https://github.com/virattt)가 만든 AI 기반 헤지펀드의 개념 증명(Proof of Concept) 프로젝트입니다. 이 프로젝트는 **교육 및 연구 목적**으로 제작되었으며, 실제 거래나 투자에 사용하기 위한 용도는 아닙니다.

본 분석은 공식 README, 소스 코드 구조(`src/` 디렉토리) 및 관련 자료를 종합하여 작성

> **참고 링크**:  
> - GitHub 저장소: [https://github.com/virattt/ai-hedge-fund](https://github.com/virattt/ai-hedge-fund)

## 컨셉 (Concept)

### 핵심 아이디어

**여러 명의 AI 에이전트(AI Agent)가 각자의 투자 철학과 전문성을 바탕으로 주식을 분석하고, 포트폴리오 매니저가 최종 투자 결정을 내리는 가상의 헤지펀드 시스템을 시뮬레이션하는 것**. 각 에이전트는 워런 버핏, 찰리 멍거, 캐시 우드 등 실제 전설적인 투자자들의 투자 전략을 모방하도록 설계되었습니다.

> 💡 "헤지펀드"라는 이름이 붙었지만, 실제로 돈을 굴리는 시스템이 아니라 AI가 투자 분석을 어떻게 수행할 수 있는지 실험하고 학습하기 위한 **교육용 플랫폼**이라는 점이 가장 중요합니다.

### 작동 흐름

1. 사용자가 분석하고 싶은 종목(예: AAPL, MSFT, NVDA 등)을 입력  
2. 여러 AI 에이전트들이 동시에 해당 종목을 각자의 투자 철학과 분석 기준에 따라 평가  
3. 각 에이전트는 "매수(BUY)", "매도(SELL)", "보유(HOLD)" 같은 트레이딩 신호를 생성  
4. 리스크 매니저가 위험도를 계산하여 포지션 한도 설정  
5. 포트폴리오 매니저 에이전트가 모든 분석 결과를 종합하여 최종 주문(거래 결정안) 생성  

실제 매매는 실행되지 않고, 의사 결정 과정만 시뮬레이션합니다.

## 특징 (Key Features)

### 1. 다중 AI 에이전트 협업 구조

이 프로젝트의 가장 큰 특징은 **여러 개의 AI 에이전트가 체계적으로 협력**하는 멀티 에이전트 시스템입니다.

**투자 전략 기반 에이전트 (실력파 투자자 14인 모방):**

| 에이전트 | 모델링한 투자자 | 투자 철학 |
|---|---|---|
| 벤 그레이엄 (Ben Graham) | "가치 투자의 아버지" | 오직 안전마진을 갖춘 숨은 가치주만 매수 |
| 워런 버핏 (Warren Buffett) | "오마하의 현인" | 훌륭한 기업을 적정 가격에 매수, 경쟁 우위와 경영진의 질을 중요시 |
| 찰리 멍거 (Charlie Munger) | 버크셔 해서웨이 부회장 | 훌륭한 사업을 공정한 가격에 매수하는 데 집중, 특히 사업 모델의 질에 무게 |
| 빌 애크먼 (Bill Ackman) | 행동주의 투자자 | 과감한 포지션을 취하고 기업 변화를 주도 |
| 캐시 우드 (Cathie Wood) | "성장 투자의 여왕" | 혁신과 파괴적 기술의 힘을 신뢰 |
| 필립 피셔 (Phil Fisher) | 전설적 성장주 투자자 | 스커틀벗 분석(현장 조사)을 통한 성장주 발굴 |
| 스탠리 드러켄밀러 (Stanley Druckenmiller) | 거시 경제 투자 전설 | 비대칭적 기회와 성장 잠재력을 추구 |

**전문 분석 에이전트:**

- **가치 평가 에이전트 (Valuation Agent):** 주식의 내재 가치를 계산하고 트레이딩 신호를 생성  
- **기술 분석 에이전트 (Technicals Agent):** 이동평균 등 기술적 지표 분석 및 신호 생성  
- **펀더멘털 분석 에이전트 (Fundamentals Agent):** 기업의 재무제표, 수익성 등 기본 데이터 분석  
- **시장 심리 분석 에이전트 (Sentiment Agent):** 뉴스와 시장 분위기를 분석하여 신호 생성  
- **리스크 매니저 (Risk Manager):** 리스크 지표 계산 및 포지션 한도 설정  
- **포트폴리오 매니저 (Portfolio Manager):** 최종 거래 결정 및 주문 생성  

### 2. 대규모 언어 모델(LLM) 기반 분석

LLM을 사용하여 금융 데이터를 분석하고 트레이딩 결정을 내립니다.

지원하는 LLM 제공업체:
- **OpenAI** (GPT-4o, GPT-4o-mini 등) - API 키 필요  
- **Groq** (DeepSeek, Llama 3 등) - API 키 필요  
- **Anthropic** (Claude) - API 키 필요  
- **DeepSeek** - API 키 필요  
- **로컬 모델** (Ollama를 통한 오픈소스 LLM) - 인터넷 연결 없이 실행 가능  

### 3. 백테스팅(Backtesting) 기능

과거 데이터로 전략의 성과를 시뮬레이션할 수 있습니다.

```bash
poetry run python src/backtester.py --ticker AAPL --start-date 2024-01-01 --end-date 2024-03-01
```
### 4. API 서버 및 웹 인터페이스

app/backend 폴더에 FastAPI 프레임워크를 사용하여 RESTful API 서버를 구축할 수 있습니다. app/frontend 폴더를 통해 웹 기반 인터페이스도 제공되어, CLI 대신 시각적 환경에서 프로젝트를 실행할 수 있습니다.

###  5. 구조화된 아키텍처

LangGraph를 활용하여 복잡한 의사 결정 과정을 체계적으로 관리하고 있습니다.

###  6. 무료 금융 데이터 활용
AAPL, GOOGL, MSFT, NVDA, TSLA와 같은 주요 종목에 대해서는 API 키 없이도 무료로 금융 데이터를 이용할 수 있습니다. 다른 종목 분석 시에는 Financial Datasets API 키가 필요합니다.

###  7. 에이전트 추론 과정 시각화
--show-reasoning 플래그를 추가하면 각 에이전트가 어떤 근거로 판단했는지 그 사고 과정을 콘솔에 출력할 수 있어, 분석 과정을 투명하게 확인할 수 있습니다.

###  한계 및 주의사항
교육용 한정: 실제 트레이딩이나 투자 목적으로 사용할 수 없으며, 금전적 손실에 대한 책임은 사용자에게 없음을 명시하고 있습니다.

과거 성과와 미래 성과의 무관함: 과거 데이터 기반 백테스트 결과가 미래 수익률을 보장하지 않습니다.

비용 문제: OpenAI, Anthropic 등 클라우드 LLM 사용 시 API 호출 비용이 발생할 수 있습니다. 이 문제를 해결하기 위해 로컬 Ollama LLM을 사용할 수 있는 기능도 제공됩니다.

데이터 의존성: 정확한 분석을 위해 외부 금융 데이터 API에 의존하며, 무료 제공 종목 외의 데이터 사용 시 유료 API 키가 필요합니다.

실제 실행 불가: 실제 매매 주문이 체결되지는 않습니다. 이는 시뮬레이션 및 연구용 시스템입니다.

###  활용 가치와 시사점
이 프로젝트는 AI와 금융 기술의 접목 가능성을 보여주는 훌륭한 예시입니다. 실제 세계에서는 Point72, Citadel, Renaissance Technologies 같은 헤지펀드들도 AI와 머신러닝을 활용한 퀀트 전략을 적극 도입하고 있습니다. ai-hedge-fund 기본 프레임워크를 확장하거나 변형하여, 다음과 같은 학습 용도로 활용해볼 수 있습니다:

AI의 금융 데이터 분석 능력 이해 및 실험

LLM을 활용한 복잡한 의사 결정 시스템 설계 방법 학습

LangGraph를 활용한 멀티 에이전트 시스템 구축 경험

AI 기반 투자 전략의 프로토타이핑
이 모든 것은 실제 투자보다는 교육과 연구의 맥락에서 이루어져야 함을 기억하는 것이 중요합니다.
